Sintaxe do @keyframes

Aluno 1: Como escrever um @keyframes — from/to vs porcentagens.

from / to
0% 25% 50% 75% 100%
0%, 100% iguais
= loop suave 
/* from/to: atalho pra 0% e 100% — bom pra animações simples */
@keyframes mover {
    from { transform: translateX(-40px); }  /* ponto de início */
    to   { transform: translateX( 40px); }  /* ponto de fim */
}

/* porcentagens: controla cada etapa do meio */
@keyframes etapas {
    0%   { transform: translateX(-40px); background: green;  }
    25%  { transform: translateY(-40px); background: yellow; }
    50%  { transform: translateX( 40px); background: red;    }
    100% { transform: translateX(-40px); background: green;  }
}

/* 0% e 100% iguais = loop sem "salto" */
@keyframes pulso {
    0%, 100% { transform: scale(1);   }
    50%      { transform: scale(1.5); }
}

/* animation-direction: pra onde a animação vai */
animation-direction: normal;            /* padrão: sempre de 0% → 100% */
animation-direction: reverse;           /* sempre de 100% → 0% */
animation-direction: alternate;         /* alterna: → ← → ← (sem salto!) */
animation-direction: alternate-reverse; /* alterna: ← → ← → */

/* animation-fill-mode: o que acontece antes/depois de rodar */
animation-fill-mode: none;        /* padrão: volta ao estado original */
animation-fill-mode: forwards;    /* FICA no estado final quando termina */
animation-fill-mode: backwards;   /* aplica o estado inicial durante o delay */
animation-fill-mode: both;        /* combina os dois — mais usado! */

/* exemplo clássico: aparece e FICA aparecido */
.aparecer {
    opacity: 0;
    animation: fadeIn 0.5s ease forwards;  /* sem forwards voltaria pra 0! */
}

/* animation-play-state: pausa ou retoma */
.spinner {
    animation: rodar 1s linear infinite;
    animation-play-state: running;   /* padrão: rodando */
}
.spinner:hover {
    animation-play-state: paused;    /* pausa ao passar o mouse */
}
/* via JS: elemento.style.animationPlayState = "paused" */

/* múltiplas animações: separa por vírgula */
.elemento {
    animation:
        vai-volta  2s  ease-in-out  infinite  alternate,  /* move */
        troca-cor  3s  linear       infinite  normal,     /* cor */
        pulsa      0.8s ease        infinite  alternate;  /* tamanho */
}

/* ATALHO COMPLETO — ordem das propriedades: */
/* animation: nome  duração  timing  delay  iterações  direção  fill  play-state */
animation: pular 1.2s ease 0.2s infinite alternate both running;

Aluno 6: O transform é a propriedade mais usada em animações. São 4 funções principais.

translate
move
rotate
gira
scale
tamanho

entorta
combinado
os 3 juntos 
/* translate: move o elemento sem afetar o layout ao redor */
transform: translateX(50px);         /* move pra direita */
transform: translateY(-30px);        /* move pra cima (Y negativo = cima) */
transform: translate(50px, -30px);   /* move nos dois eixos de uma vez */

/* rotate: gira em torno do próprio centro */
transform: rotate(45deg);   /* 45 graus */
transform: rotate(360deg);  /* volta completa */

/* scale: muda o tamanho. 1 = normal, 0.5 = metade, 2 = dobro */
transform: scale(1.5);    /* 50% maior, nos dois eixos */
transform: scaleX(2);    /* só estica na horizontal */

/* skew: entorta como um paralelogramo */
transform: skewX(20deg);
transform: skew(20deg, 5deg);

/* combinando: escreve tudo na mesma linha, separado por espaço */
transform: translateX(40px) rotate(180deg) scale(1.3);

Performance e Boas Práticas

Aluno 7: O que o browser faz por dentro, quais propriedades custam caro, e como não travar a animação.

transform + opacity
CPU ✗ width + margin 
/* ─── 1. PIPELINE DE RENDERIZAÇÃO ────────────────────────────────
   O browser passa por 3 etapas a cada frame (60fps = 16ms por frame):
   Layout → Paint → Composite

   Propriedades que causam LAYOUT (mais lento — recalcula tudo):
   width, height, margin, padding, top, left, font-size...

   Propriedades que causam só PAINT (médio):
   background, color, border-color, box-shadow...

   Propriedades que só fazem COMPOSITE (mais rápido — vai pra GPU):
   transform, opacity  ← SEMPRE prefira essas nas animações! */

/* ─── 2. WILL-CHANGE ──────────────────────────────────────────────
   Avisa o browser com antecedência pra criar uma camada separada na GPU.
   Use só quando necessário — cada camada consome memória de vídeo. */
.elemento-animado {
    will-change: transform, opacity;   /* cria a camada antes da animação */
}
/* remova via JS depois que a animação acabar: */
/* el.style.willChange = 'auto'; */

/* ─── 3. PREFERS-REDUCED-MOTION ───────────────────────────────────
   Acessibilidade obrigatória: usuários com epilepsia, vertigem ou
   sensibilidade visual podem ativar esta preferência no SO. */
@media (prefers-reduced-motion: reduce) {
    * {
        animation-duration:   0.001ms !important;
        animation-iteration-count: 1 !important;
        transition-duration:    0.001ms !important;
    }
}

/* ─── 4. COMPOSIÇÃO DO ATALHO — ordem importa! ───────────────────
   Quando há 2 valores de tempo, o browser interpreta assim:
   1° = duration   2° = delay
   animation: nome duration timing-function delay count direction fill play-state */
.correto   { animation: fade 1s ease 0.5s; } /* dur=1s delay=0.5s ✓ */
.invertido { animation: fade 0.5s ease 1s; } /* dur=0.5s delay=1s  ✓ */

/* ─── 5. CONTAINMENT ─────────────────────────────────────────────
   Isola o impacto de layout de um elemento pai — nada vaza pra fora. */
.card { contain: layout style; }

/* ─── SCROLL-DRIVEN: BARRA DE PROGRESSO NATIVA ─────────────────
   animation-timeline: scroll()  → usa o scroll do elemento como timeline
   animation-range     → define entre quais posições de scroll a animação vai  */

.progress-bar {
    position: fixed;
    top: 0; left: 0;
    height: 4px;
    background: #0056b3;
    transform-origin: left;

    /* conecta ao scroll — nenhum JS necessário! */
    animation: progresso linear;
    animation-timeline: scroll();   /* scroll() = root scroller */
}
@keyframes progresso {
    from { transform: scaleX(0); }
    to   { transform: scaleX(1); }
}

/* parâmetros de scroll() ─────────────────────────────────────────
   scroll()                → block direction, nearest scroller
   scroll(root)            → usa a janela inteira
   scroll(self)            → o próprio elemento
   scroll(inline)          → eixo horizontal
   scroll(root block)      → janela, eixo vertical (explícito)    */

/* ─── SCROLL-DRIVEN: REVELAR AO ENTRAR NA VIEWPORT ─────────────
   view() → timeline baseada em quando o elemento aparece na tela  */

.card {
    opacity: 0;
    animation: revelar linear both;
    animation-timeline: view();
    /* entry 0%  = elemento acabou de entrar na viewport (borda inferior)
       entry 100% = elemento está totalmente visível                     */
    animation-range: entry 0% entry 40%;
}
@keyframes revelar {
    from { opacity: 0; transform: translateY(30px); }
    to   { opacity: 1; transform: translateY(0);     }
}

/* animation-range: intervalos possíveis ─────────────────────────
   cover   → cobre toda a passagem pelo scroller
   entry   → entra na viewport
   exit    → sai da viewport
   contain → enquanto está totalmente dentro

   Exemplos:
   animation-range: cover 0% cover 100%   → toda a passagem
   animation-range: entry 0% entry 50%    → primeira metade da entrada
   animation-range: exit 0% exit 100%     → enquanto sai               */
